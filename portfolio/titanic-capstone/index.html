Titanic Capstone Project

A machine learning capstone project using the Titanic dataset. The project demonstrates the complete ML lifecycle, from data processing and model training to deployment in an interactive web interface with persistent storage.

About this project

This project is a full-stack machine learning application built around the Titanic dataset, with the goal of predicting whether a passenger survived based on demographic and travel-related features. The project was developed collaboratively using Git and Scrum practices, simulating a real-world development workflow.

The system covers the entire machine learning pipeline:

  • Data cleaning and preprocessing
  • Feature engineering
  • Model training and evaluation
  • Deployment within a web application
  • Persistent storage of predictions

On the machine learning side, we worked with pandas and scikit-learn to prepare the dataset and train models. We implemented both a Logistic Regression baseline model (~81.6% accuracy) and a Random Forest model (~83.2% accuracy), ultimately selecting Random Forest due to its ability to capture non-linear relationships.

My contributions focused on:

  • Preparing and transforming data for model training (handling missing values, encoding categorical variables)
  • Designing and implementing feature engineering strategies such as AgeGroup, FareGroup, FamilySize, and IsAlone
  • Integrating the trained model into a Django backend, enabling real-time predictions based on user input
  • Structuring the application to connect frontend forms, backend logic, and ML inference seamlessly
  • Contributing to testing and validation of the system

The final product is a working web application where users can input passenger data, receive predictions instantly, and view previous results stored in a database.

Design choices

The technical design of this project was centered around creating a clear separation between the machine learning pipeline and the web application logic, while ensuring smooth integration between the two. The system follows a full-stack architecture where Django handles request processing, validation, and user interaction, while the machine learning model operates as a dedicated prediction layer. This separation makes the system easier to maintain, extend, and debug.

On the data side, preprocessing decisions were made to balance simplicity and performance. Missing values were handled using straightforward imputation techniques, and categorical variables were encoded numerically to ensure compatibility with the models. Feature engineering played a key role in improving model performance while keeping interpretability high; for example, grouping age and fare into categories helped capture non-linear patterns without overcomplicating the dataset.

From a backend perspective, the application was designed to handle the full prediction flow: validating user input, transforming it into model-ready features, generating predictions, and storing results in a SQLite database. The use of Django models ensured data integrity, while views and forms structured the request–response cycle cleanly.

Testing was also an important part of the design. Unit tests were implemented for forms, models, views, and the full prediction workflow, ensuring that both individual components and the system as a whole behave reliably. Overall, the design emphasizes robustness, clarity, and real-world applicability, demonstrating how machine learning models can be effectively deployed within a production-like web environment.

Model development & analysis
Feature Engineering Code (AgeGroup, FareGroup, FamilySize, IsAlone)

This code shows the feature engineering process used to transform raw passenger data into meaningful input variables for the model. Continuous variables such as age and fare were grouped into categorical ranges to capture non-linear survival patterns, while new features like FamilySize and IsAlone were created to represent social dynamics. These transformations improved both model interpretability and predictive performance.

Title Extraction from Name Column

This snippet demonstrates how titles were extracted from passenger names and converted into a structured feature. Titles such as Mr, Mrs, and Miss provide indirect information about social status, gender, and age, making them valuable predictors for survival. The transformation simplifies textual data into a format usable by machine learning models.

Survival Patterns by Age, Fare, and Class

This combined visualization shows how survival probability varies across age groups, fare categories, and passenger classes. Children and first-class passengers had significantly higher survival rates, while lower fare groups and third-class passengers were more vulnerable. These patterns highlight the impact of both social status and evacuation priorities, supporting the use of AgeGroup and FareGroup as engineered features.

Age Distribution and Family Size Impact on Survival

This visualization combines age distribution and family size analysis to show how these factors influence survival. Younger passengers were more likely to survive, while survival rates varied depending on whether passengers traveled alone or in groups. Medium-sized families had better outcomes compared to individuals or very large groups, which motivated the creation of FamilySize and IsAlone features.

Model Evaluation (Random Forest vs Logistic Regression)

This screenshot presents evaluation metrics for both models, including accuracy, precision, recall, and F1-score. While Logistic Regression provides a strong and interpretable baseline, Random Forest achieves slightly higher accuracy and better captures complex patterns, leading to its selection as the final model.
